# PennyPrompt v0.1.0-alpha.5

Publication status: **prepared, not yet published.** This cut is intended to be published as a
GitHub prerelease (`prerelease: true`, intentional alpha policy) once the tag is pushed.

## Summary

Phase A — **compatibility & cost accuracy.** This release makes the headline promise ("works with
your agent, zero changes") literally true for Anthropic-native agents, and makes reported cost
correct on the flagship cache-heavy coding-agent workload.

Highlights:

- **Native Anthropic ingress.** The proxy now accepts `POST /v1/messages` in the native Anthropic
  Messages format (system prompts, tools, and `tool_use`/`tool_result` content blocks preserved),
  returns the native Anthropic response shape, and forwards the native streaming SSE sequence
  unmodified while accounting usage for reconciliation. Point OpenClaw / claw-code at
  `ANTHROPIC_BASE_URL=http://localhost:8585` — no translation shim.
- **Cache-accurate cost.** Prompt-cache read and write tokens are read from Anthropic and OpenAI
  usage and priced with dedicated rates. Reports break usage into fresh input / cache read /
  cache write / output, and totals reconcile to the ledger.
- **Guardrails.** An inbound concurrency limit and a configurable upstream timeout protect the
  single SQLite writer and stop a hung provider from pinning a request.
- **Brand alignment.** The shipped binary is now `pennyprompt` (a `penny-cli` compatibility symlink
  ships for one train).

## Upgrade Notes

- The command is now `pennyprompt`. The installer creates a `penny-cli` symlink and prints a
  deprecation notice on legacy use; the symlink is removed in `beta.1`. Pinned installs of
  pre-rename tags (`v0.1.0-alpha.4` and older) still resolve via the legacy `penny-cli-*` asset.
- Migrations `0010`/`0011` are additive and apply automatically on startup. `request_usage.input_tokens`
  now records the fresh (non-cached) input count; cached read/write tokens are recorded separately.
- New config fields `[server].max_inflight_requests` (default 64) and `[server].upstream_timeout_ms`
  (default 60000) are optional; existing configs keep working with the defaults.

  renders the cache breakdown with a reconciled cost.

## Known Limitations

Current alpha limitations are documented in [docs/LIMITATIONS.md](../LIMITATIONS.md).

Important: admin APIs do not implement bearer/admin-token authentication. Use a Unix socket or
loopback TCP bind only; do not expose the admin plane to LAN or public networks.
